# 구글 시트 저장 전환 설계

## 목표

특별실 예약 웹앱의 저장 방식을 노드 서버 파일 저장에서 구글 시트 저장으로 바꾼다. 사용자는 노드 서버 주소를 몰라도 되고, 정적 웹주소만 열어 예약 조회, 예약 추가, 예약 삭제를 할 수 있어야 한다.

## 배경

현재 앱은 노드 서버의 `/api/reservations`와 `data/reservations.json`에 의존한다. 이 구조는 로컬 실행에서는 동작하지만, 깃허브 저장소 화면이나 정적 배포 주소에서는 예약 저장을 할 수 없다. 학교 현장에서는 여러 선생님이 같은 웹주소만 열어 쓰는 방식이 더 적합하다.

## 접근 방식

구글 시트를 예약 저장소로 쓰고, 구글 앱스 스크립트를 작은 저장 API로 쓴다.

- 화면 앱은 정적 웹앱으로 배포한다.
- 예약 자료는 구글 시트 행으로 저장한다.
- 구글 앱스 스크립트 웹앱이 조회, 생성, 삭제 요청을 처리한다.
- 노드 서버는 운영 필수 요소에서 제거한다.

## 구글 시트 구조

시트 이름은 `reservations`로 한다.

첫 행은 다음 열 이름을 가진다.

- `id`
- `date`
- `period`
- `room`
- `grade`
- `classNumber`
- `passwordHash`
- `createdAt`

예약 목록 조회 응답에는 `passwordHash`를 포함하지 않는다.

## 특별실 목록

특별실은 다음 여섯 개로 고정한다.

- 창의놀이실
- 청계누리(강당)
- 컴퓨터실(4층)
- AI실(2층)
- 음악실
- 다모임실

구글 앱스 스크립트도 같은 목록만 허용한다.

## 구글 앱스 스크립트 API

구글 앱스 스크립트 웹앱 주소 하나를 사용한다. 화면 앱은 이 주소로 요청한다.

요청은 모두 JSONP 방식의 `GET`으로 보낸다. 정적 웹앱에서 구글 앱스 스크립트 응답을 안정적으로 읽기 위해 `callback`과 `payload` 쿼리 값을 사용한다.

### 예약 목록 조회

요청:

```txt
?callback=콜백함수이름&payload={"action":"list"}
```

응답:

```json
{
  "ok": true,
  "reservations": []
}
```

### 예약 생성

요청:

```txt
?callback=콜백함수이름&payload={"action":"create","reservation":{"date":"2026-06-10","period":1,"room":"창의놀이실","grade":1,"classNumber":1,"password":"1234"}}
```

처리:

- 필수 값이 모두 있는지 확인한다.
- 날짜, 교시, 특별실이 같은 예약이 이미 있으면 거부하고 기존 예약 학급을 알려준다.
- 예약 날짜가 오늘부터 8주 뒤 범위를 벗어나면 거부한다.
- 선택한 학년에 없는 반이면 거부한다.
- 삭제 비밀번호는 그대로 저장하지 않고 앱스 스크립트의 해시 함수로 만든 확인값으로 저장한다.
- 생성된 예약은 `passwordHash` 없이 응답한다.

### 예약 삭제

요청:

```txt
?callback=콜백함수이름&payload={"action":"delete","id":"예약 식별값","password":"1234"}
```

처리:

- 식별값으로 행을 찾는다.
- 삭제 비밀번호 확인값과 입력값을 비교한다.
- 삭제 비밀번호 또는 관리자 비밀번호가 맞으면 행을 삭제한다.

## 화면 앱 변경

`src/api.js`는 기존 `/api/reservations` 호출 대신 구글 앱스 스크립트 웹앱 주소를 호출한다.

웹앱 주소는 코드에 직접 박지 않고 환경값으로 둔다.

- 환경값 이름: `VITE_GOOGLE_SCRIPT_URL`
- 값이 없으면 화면에 설정 안내를 보여준다.

예약 화면, 주간 시간표, 필터, 삭제 흐름은 그대로 유지한다. 화면에는 날짜별 목록 보기, 오늘 날짜 강조, 특별실별 색상, 저장소 연결 상태를 추가한다.

## 구글 앱스 스크립트 파일

저장소에 앱스 스크립트 코드를 문서와 함께 둔다.

- `google-apps-script/Code.gs`
- `google-apps-script/README.md`

`Code.gs`는 구글 앱스 스크립트 편집기에 붙여 넣어 배포할 수 있는 완성 코드여야 한다.

삭제 비밀번호 확인값은 앱스 스크립트 기본 유틸리티로 만든다. 외부 라이브러리를 쓰지 않는다.

## 오류 처리

- 구글 앱스 스크립트 주소가 설정되지 않으면 설정 안내를 보여준다.
- 구글 앱스 스크립트가 오류를 응답하면 해당 오류 메시지를 보여준다.
- 네트워크 실패는 연결 오류로 안내한다.
- 중복 예약은 기존 중복 예약 메시지를 유지한다.
- 삭제 비밀번호 실패는 기존 삭제 실패 메시지를 유지한다.
- 관리자 비밀번호는 스크립트 속성 `ADMIN_DELETE_PASSWORD`로 설정한다.

## 운영 절차

1. 구글 시트를 만든다.
2. 첫 시트 이름을 `reservations`로 바꾼다.
3. 첫 행에 열 이름을 입력한다.
4. 구글 앱스 스크립트 편집기에 `Code.gs` 내용을 붙여 넣는다.
5. 시트 식별값을 스크립트 설정에 넣는다.
6. 관리자 삭제를 쓸 경우 관리자 비밀번호를 스크립트 설정에 넣는다.
7. 웹앱으로 배포한다.
8. 배포된 웹앱 주소를 `VITE_GOOGLE_SCRIPT_URL`에 넣고 화면 앱을 배포한다.

## 확인 기준

- 정적 웹앱 주소에서 예약 목록을 불러올 수 있다.
- 정적 웹앱 주소에서 예약을 생성하면 구글 시트에 행이 추가된다.
- 같은 날짜, 교시, 특별실 중복 예약은 거부된다.
- 학년별 반 수 제한이 적용된다.
- 오늘부터 8주 뒤까지만 예약할 수 있다.
- 올바른 삭제 비밀번호로 예약을 삭제하면 구글 시트 행이 삭제된다.
- 관리자 비밀번호로 예약을 삭제하면 구글 시트 행이 삭제된다.
- 틀린 삭제 비밀번호로는 삭제되지 않는다.
- 노드 서버를 실행하지 않아도 정적 웹앱에서 저장 기능이 동작한다.

## 제외

- 교사 구글 로그인
- 관리자 승인
- 구글 시트 자동 생성
- 반복 예약
- 알림 발송
