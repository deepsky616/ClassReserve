# 특별실 예약 웹앱 설계

## 목표

초등학교와 유치원에서 특별실 이용을 교시 단위로 간편하게 예약하고, 모든 사용자가 같은 예약 결과를 주간 시간표 형태로 볼 수 있는 웹앱을 만든다.

첫 버전은 로그인 없이 빠르게 쓰는 것을 우선한다. 예약자는 유치원 또는 초등학교 학년과 반, 삭제 비밀번호를 입력해 예약하고, 같은 특별실의 같은 날짜와 교시에 중복 예약이 생기지 않도록 막는다.

## 범위

### 포함

- 모든 사용자가 주간 예약표 조회
- 날짜, 교시, 특별실, 유치원 또는 초등학교 학년, 반으로 예약 추가
- 같은 날짜, 같은 교시, 같은 특별실 중복 예약 방지와 기존 예약 학급 안내
- 예약할 때 입력한 삭제 비밀번호로 예약 삭제
- 특별실 필터
- 이전 주, 이번 주, 다음 주 이동
- 서버 파일에 예약 자료 저장

### 제외

- 교사 계정 로그인
- 관리자 승인
- 알림 발송
- 반복 예약
- 학사 일정 연동
- 실제 데이터베이스 저장

## 기본 설정

### 특별실

- 창의놀이실
- 청계누리(강당)
- 컴퓨터실(4층)
- AI실(2층)
- 음악실
- 다모임실

### 교시

- 1교시
- 2교시
- 3교시
- 4교시
- 5교시
- 6교시

### 학년과 반

- 학년은 유치원 또는 1학년부터 6학년까지 선택한다.
- 반은 1반부터 10반까지 선택한다.
- 반 범위는 나중에 설정값으로 바꿀 수 있게 둔다.

## 화면 설계

첫 화면은 이번 주 예약표다.

상단에는 다음 조작을 둔다.

- 이전 주
- 이번 주
- 다음 주
- 특별실 필터

본문은 주간 시간표다.

- 가로축은 월요일부터 금요일까지다.
- 세로축은 1교시부터 6교시까지다.
- 각 칸에는 해당 날짜와 교시에 예약된 특별실, 학년, 반을 표시한다.
- 특별실 필터가 선택되면 해당 특별실 예약만 표시한다.

예약 입력 영역은 같은 화면에 둔다.

- 날짜 선택
- 교시 선택
- 특별실 선택
- 학년 선택
- 반 선택
- 삭제 비밀번호 입력
- 예약하기 단추

예약 항목에는 삭제 단추를 둔다. 삭제 단추를 누르면 비밀번호를 입력받고, 예약할 때 입력한 비밀번호와 맞으면 삭제한다.

## 자료 구조

예약 하나는 다음 값을 가진다.

- 고유 식별값
- 날짜
- 교시
- 특별실
- 학년
- 반
- 삭제 비밀번호 확인값
- 만든 시각

서버는 예약 목록을 `data/reservations.json` 파일에 저장한다. 저장 파일은 배열 형태로 유지한다.

삭제 비밀번호는 예약 목록 조회 결과에 포함하지 않는다. 저장 파일에도 입력값 그대로 저장하지 않고, 서버가 비교할 수 있는 확인값만 저장한다.

## 서버 기능

서버는 세 가지 기능을 제공한다.

- 예약 목록 조회
- 예약 생성
- 예약 삭제

예약 생성 요청을 받으면 서버는 먼저 필수 값이 모두 있는지 확인한다. 그다음 같은 날짜, 같은 교시, 같은 특별실 예약이 이미 있는지 확인한다. 이미 있으면 예약을 저장하지 않고 중복 안내를 돌려준다.

예약 삭제 요청을 받으면 서버는 예약 식별값과 삭제 비밀번호를 확인한다. 비밀번호가 맞으면 예약을 삭제하고, 틀리면 삭제하지 않는다.

## 오류 처리

- 필수 입력이 빠진 경우 입력 안내를 보여준다.
- 이미 예약된 경우 중복 예약 안내를 보여준다.
- 삭제 비밀번호가 틀린 경우 삭제 실패 안내를 보여준다.
- 서버 저장 파일을 읽거나 쓸 수 없는 경우 저장 오류 안내를 보여준다.

## 기술 구조

저장소에는 하나의 웹앱과 작은 서버를 함께 둔다.

- 화면은 리액트 기반으로 만든다.
- 빠른 개발 서버 도구를 사용한다.
- 서버는 노드 기반으로 만든다.
- 예약 자료는 서버의 파일 저장 모듈을 통해 읽고 쓴다.

개발 실행은 한 명령으로 화면과 서버를 함께 띄우는 방식을 목표로 한다.

## 확인 기준

다음이 동작하면 첫 버전은 완료로 본다.

- 앱을 실행하면 이번 주 예약표가 보인다.
- 날짜, 교시, 특별실, 학년, 반, 삭제 비밀번호로 예약할 수 있다.
- 모든 사용자가 같은 예약 목록을 볼 수 있다.
- 같은 날짜, 같은 교시, 같은 특별실은 중복 예약할 수 없다.
- 올바른 삭제 비밀번호로 예약을 삭제할 수 있다.
- 틀린 삭제 비밀번호로는 예약을 삭제할 수 없다.
- 특별실 필터와 주 이동이 동작한다.

## 나중에 확장할 수 있는 항목

- 교사 계정 로그인
- 학년과 반 설정 화면
- 특별실 추가와 숨김
- 반복 예약
- 실제 데이터베이스 저장
- 예약 내보내기
