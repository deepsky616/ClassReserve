# 구글 시트 저장소 설정

이 폴더의 `Code.gs`를 구글 앱스 스크립트에 붙여 넣으면 특별실 예약 자료를 구글 시트에 저장할 수 있습니다.

## 1. 구글 시트 만들기

1. 새 구글 시트를 만듭니다.
2. 첫 번째 시트 이름을 `reservations`로 바꿉니다.
3. 첫 행에 아래 열 이름을 입력합니다.

```txt
id	date	period	room	grade	classNumber	passwordHash	createdAt
```

4. 주소에서 시트 식별값을 복사합니다.

예시 주소:

```txt
https://docs.google.com/spreadsheets/d/시트식별값/edit
```

## 2. 앱스 스크립트 만들기

1. 구글 시트에서 `확장 프로그램` 메뉴를 엽니다.
2. `Apps Script`를 선택합니다.
3. 기본 파일 내용을 지우고 `google-apps-script/Code.gs` 내용을 붙여 넣습니다.
4. 왼쪽의 `프로젝트 설정`을 엽니다.
5. `스크립트 속성`에 다음 값을 추가합니다.

```txt
이름: SPREADSHEET_ID
값: 복사한 시트 식별값
```

## 3. 웹앱 배포

1. 오른쪽 위 `배포`를 누릅니다.
2. `새 배포`를 선택합니다.
3. 유형은 `웹 앱`을 선택합니다.
4. 실행 권한은 `나`로 선택합니다.
5. 접근 권한은 학교 상황에 맞게 선택합니다.
   - 학교 계정 안에서만 쓸 경우: 조직 사용자
   - 링크만 있으면 누구나 쓰게 할 경우: 모든 사용자
6. 배포 후 웹앱 주소를 복사합니다.

주소는 보통 아래 형태입니다.

```txt
https://script.google.com/macros/s/배포식별값/exec
```

## 4. 화면 앱 연결

프로젝트 루트에 `.env` 파일을 만들고 아래처럼 입력합니다.

```txt
VITE_GOOGLE_SCRIPT_URL=https://script.google.com/macros/s/배포식별값/exec
```

그다음 화면 앱을 빌드하거나 배포합니다.

```bash
npm run build
```

개발 중에는 아래 명령으로 확인합니다.

```bash
npm run dev
```

## 확인 방법

1. 앱스 스크립트 편집기에서 직접 실행할 때는 `doGet`이 아니라 `testListReservations`를 선택합니다.
2. `testListReservations` 실행 결과가 배열이면 시트 연결이 된 것입니다.
3. 웹앱을 엽니다.
4. 예약을 하나 만듭니다.
5. 구글 시트에 새 행이 추가되는지 확인합니다.
6. 같은 날짜, 같은 교시, 같은 특별실로 다시 예약해 중복 안내가 나오는지 확인합니다.
7. 중복 안내에 이미 예약한 학급이 표시되는지 확인합니다.
8. 유치원 예약이 저장되는지 확인합니다.
9. 올바른 삭제 비밀번호로 삭제되는지 확인합니다.

## 주의

- 앱스 스크립트 편집기에서 `doGet`을 직접 실행하면 실제 웹 요청 정보가 없어서 정상 테스트가 아닙니다.
- `Code.gs`를 수정한 뒤에는 새 버전으로 다시 배포해야 실제 웹앱에 반영됩니다.
- 삭제 비밀번호는 원문으로 저장하지 않고 해시 값으로 저장됩니다.
- 삭제 비밀번호를 잊은 경우 담당자가 구글 시트에서 해당 예약 행을 직접 삭제해야 합니다.
- 구글 시트 권한이나 앱스 스크립트 배포 권한이 막혀 있으면 예약 저장이 되지 않습니다.
