# 특별실 예약

초등학교와 유치원에서 특별실을 교시 단위로 예약하는 간편 웹앱입니다.

## 기능

- 주간 시간표 형태로 예약 조회
- 유치원은 반 없이, 초등학교는 학년과 반으로 예약
- 시작 교시와 끝 교시를 골라 여러 교시를 한 번에 예약
- 같은 날짜, 같은 교시, 같은 특별실 중복 예약 방지와 기존 예약 학급 안내
- 예약할 때 입력한 삭제 비밀번호로 예약 삭제
- 같은 날짜의 여러 예약 선택 삭제
- 관리자 비밀번호로 잘못된 예약 삭제
- 관리자 비밀번호로 요일별 고정 사용 등록과 삭제
- 고정 사용 시간에는 일반 예약 자동 차단
- 오늘부터 8주 뒤까지 예약 가능
- 오늘 날짜 강조, 특별실별 색상, 날짜별 목록 보기
- 구글 시트 연결 상태 표시
- 특별실 필터
- 이전 주, 이번 주, 다음 주 이동

## 기본 특별실

- 창의놀이실
- 신체활동실
- AI캠퍼스
- 음악실
- 다모임실
- 체육관
- 컴퓨터실

## 학년별 반 설정

- 유치원: 반 선택 없음
- 1학년: 1반부터 5반까지
- 2학년: 1반부터 5반까지
- 3학년: 1반부터 5반까지
- 4학년: 1반부터 7반까지
- 5학년: 1반부터 7반까지
- 6학년: 1반부터 6반까지

## 설치

노드 22 이상을 권장합니다.

```bash
npm install
```

## 개발 실행

구글 시트 저장소를 연결하려면 `.env`에 앱스 스크립트 웹앱 주소를 설정합니다.

```bash
cp .env.example .env
```

`.env` 파일의 `VITE_GOOGLE_SCRIPT_URL` 값을 실제 앱스 스크립트 웹앱 주소로 바꿉니다.

```bash
npm run dev
```

화면은 보통 `http://localhost:5173`에서 열립니다.

## 구글 시트 저장소

예약 자료는 구글 시트에 저장됩니다. 설정 방법은 [google-apps-script/README.md](google-apps-script/README.md)를 참고하세요.

## 확인

```bash
npm test
```

## 여러 교시 예약

예약 화면에서 `시작 교시`와 `끝 교시`를 선택하면 해당 범위의 모든 교시가 한 번에 예약됩니다. 예를 들어 `1교시`부터 `3교시`까지 선택하면 1교시, 2교시, 3교시 예약이 함께 생성됩니다.

범위 안의 교시 중 하나라도 이미 같은 특별실에 예약되어 있으면 전체 예약이 거부되고, 먼저 예약한 학급이나 유치원이 안내됩니다.

단, `체육관`은 같은 날짜와 같은 교시에 2건까지 예약할 수 있습니다. 3번째 예약부터는 먼저 예약한 학급이나 유치원 안내와 함께 거부됩니다.

예약 저장 뒤 최신 목록까지 한 번에 돌려받는 방식이라 불필요한 추가 조회를 줄였습니다.

예약 등록, 고정 사용 등록, 예약 삭제, 고정 사용 삭제는 저장소 요청 안에서 최신 목록을 함께 계산해 반환합니다. 구글 시트를 다시 읽는 횟수와 여러 행 삭제 호출을 줄여 저장 후 화면 갱신 지연을 낮췄습니다.

## 여러 예약 선택 삭제

예약 카드의 체크박스를 선택한 뒤 `선택 삭제`를 누르면 같은 날짜의 여러 예약을 한 번에 삭제할 수 있습니다. 삭제 비밀번호 또는 관리자 비밀번호는 한 번만 입력합니다.

다른 날짜의 예약이 함께 선택되어 있으면 삭제되지 않습니다.

선택 삭제는 구글 앱스 스크립트에서 여러 예약을 한 번에 처리하고 최신 목록까지 함께 돌려받습니다.

## 고정 사용 등록

관리 도구의 `고정 사용 등록`에서 요일, 시작 교시, 끝 교시, 특별실, 사용 이름을 입력하면 매주 반복되는 고정 사용을 등록할 수 있습니다.

예를 들어 `월요일 2교시 음악실 3학년 음악`을 등록하면, 모든 월요일 2교시에 음악실 일반 예약이 막히고 시간표에는 고정 사용으로 표시됩니다.

고정 사용 등록과 삭제에는 `ADMIN_DELETE_PASSWORD`에 설정한 관리자 비밀번호가 필요합니다. 자료는 구글 시트의 `fixed_schedules` 시트에 저장됩니다.

등록된 고정 사용도 예약 카드처럼 달력에 표시됩니다. 삭제할 고정 사용은 달력에서 체크한 뒤 `고정 삭제`를 눌러 한 번에 삭제합니다. 고정 사용 설정 패널은 새 고정 사용을 등록하는 용도로만 사용합니다.

## 자료 저장

예약 자료는 연결한 구글 시트의 `reservations` 시트에 저장됩니다. 고정 사용 자료는 `fixed_schedules` 시트에 저장됩니다. 삭제 비밀번호는 그대로 저장하지 않고 확인값으로 저장되며, 예약 목록 응답에도 포함되지 않습니다.

## 삭제 비밀번호를 잊은 경우

삭제 비밀번호는 복구할 수 없습니다. 잘못 예약했는데 비밀번호를 잊은 경우에는 담당자가 구글 시트의 `reservations` 시트에서 해당 행을 확인한 뒤 삭제하는 방식으로 처리하세요.

구글 앱스 스크립트의 스크립트 속성에 `ADMIN_DELETE_PASSWORD`를 설정하면, 삭제 창에서 관리자 비밀번호를 입력해도 예약을 삭제할 수 있습니다. 비밀번호 원문을 저장하지 않는 현재 방식은 유지됩니다.
